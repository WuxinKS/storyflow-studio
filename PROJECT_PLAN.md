# StoryFlow Studio / 灵感成片 — 完整项目计划 v1

## 1. 项目定位

打造一个面向内容创作者的 **AI 导演级创作工作台**：

- 从一个想法、一段描述、一个题材、几张参考图或一段参考视频出发
- 自动生成：
  - 世界观 / 人设 / 大纲
  - 小说 / 文章 / 剧本
  - 场次拆分 / 镜头脚本 / 分镜
  - 角色与场景视觉资产
  - 配音 / 视频片段 / 成片
- 同时支持 **参考素材反向解析**，把参考图、参考视频的镜头语言、动作设计、构图风格提炼出来并迁移到原创内容中

一句话定义：

> 基于“想法 + 参考素材”，自动生成小说/剧本，并转化为可编辑分镜与视频成片的 AI 创作工作台。

## 2. 融合策略

### 2.1 源项目判断

#### MuMuAINovel 的优势
- 长文本创作能力完整
- 世界观、角色、组织、章节、伏笔等结构化写作能力强
- 适合作为 **故事引擎** 的灵感来源

#### waoowaoo 的优势
- 更接近视频/影视工作台
- 前端更现代，产品完成度更高
- 已具备异步任务、媒体资产、视频生成等基础设施
- 更适合作为 **统一产品骨架**

### 2.2 融合结论

**以 `waoowaoo` 为主骨架，吸收 `MuMuAINovel` 的长文本创作能力。**

不建议：
- 简单 iframe 拼接
- 长期双后端并行
- 双数据库长期共存

建议：
- 新建统一品牌/统一仓库/统一前端
- 先继承 `waoowaoo` 的 Next.js 工作台与任务架构
- 将 `MuMuAINovel` 的故事生成、设定管理、章节生成能力迁移为独立领域模块

## 4. 模块拆分

### 模块 1：Idea Engine（灵感输入引擎）
输入：
- 一句话创意
- 关键词
- 题材
- 风格
- 输出目标（小说 / 剧本 / 视频 / 漫画）

输出：
- 项目初始化信息
- logline
- 题材判断
- 风格方向
- 生成起点

### 模块 2：Story Engine（故事引擎）
继承 MuMuAINovel 的核心价值，负责：
- 世界观生成
- 角色设定
- 势力/组织设计
- 故事大纲
- 章节生成
- 章节润色 / 重写
- 伏笔追踪
- 文风模板

### 模块 3：Adaptation Engine（改编引擎）
负责：
- 小说 / 文章 → 剧本化改写
- 章节 → Scene 拆解
- Scene → Shot 拆解
- 对白、旁白、动作描述生成
- 每个镜头的提示词与渲染参数输出

### 模块 4：Reference Vision Engine（参考解析引擎）
负责：
- 单图分析
- 多图连续分析
- 视频抽帧与关键镜头分析
- 构图 / 景别 / 运镜 / 节奏 / 情绪 / 动作提取
- 生成可复用风格卡 / 镜头模板 / 提示词模板
- 将参考风格迁移到当前项目的 Scene/Shot

### 模块 5：Asset Engine（角色与场景资产引擎）
负责：
- 角色形象一致性管理
- 场景设定与场景图生成
- 道具 / 服装 / 状态版本化
- 资产库与复用机制

### 模块 6：Storyboard Engine（分镜引擎）
负责：
- Shot 列表管理
- Storyboard 可视化
- 每镜头参数编辑
- 参考图对照
- 镜头变体生成
- 导演视角调整

### 模块 7：Render Engine（视频生成引擎）
负责：
- 图像生成
- 视频片段生成
- 角色配音
- 字幕 / 背景音乐 / 音效
- 多镜头拼接
- 成片导出

### 模块 8：Project Workspace（项目工作台）
负责：
- 项目概览
- 阶段进度
- 任务中心
- 成本与耗时预估
- 导出中心
- 版本回溯

## 6. 后端与数据架构建议

### 6.1 推荐总体技术栈
- Next.js App Router（前后端一体）
- Prisma ORM
- Redis + BullMQ（异步任务队列）
- 对象存储（MinIO / S3）
- 后续根据需要接入 Python 微服务处理重 AI 任务

### 6.2 数据库选择建议
#### 推荐长期方案：PostgreSQL
原因：
- 更适合长文本与复杂结构化内容
- JSON / 搜索 / 向量检索扩展性更好
- 更适合复杂关系的创作系统

#### 短期 MVP 方案
- 第一阶段可保留 `waoowaoo` 现有 Prisma + MySQL 跑通 MVP
- 第二阶段视成本迁移 PostgreSQL

## 8. MVP 范围

### MVP 必须打通的闭环
1. 输入一个创意
2. 自动生成世界观 / 角色 / 大纲
3. 生成一章小说或一个短剧片段
4. 自动拆分为 Scene / Shot
5. 输出可编辑分镜列表
6. 生成静态分镜图
7. 组合导出简版视频

### MVP 可选加分项
- 上传参考图并辅助分镜优化
- 角色/场景基础一致性控制
- 异步任务中心

### MVP 暂缓内容
- 复杂多人协作
- 完整商业计费体系
- 大规模社区模板市场
- 超复杂多 Provider 编排
- 电影级后期能力

## 11. 关键风险

### 11.1 一致性风险
- 角色形象一致性
- 场景一致性
- 文本与视频一致性
- 分镜与剧情逻辑一致性

### 11.2 任务编排复杂度
- 图像/视频/配音任务慢
- 需要完善状态管理、失败重试、成本控制

### 11.3 前端复杂度
- 这是创作操作系统，不是单点工具
- 必须统一体验，避免成为“功能堆场”
